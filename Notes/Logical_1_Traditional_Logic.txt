### 1.2 THE EJOTY TECHNIQUE (LIFESAVER!)

**EJOTY = Positions at multiples of 5**
```
E = 5
J = 10
O = 15
T = 20
Y = 25

**How to use EJOTY:**
To find any letter's position, locate nearest EJOTY letter and count forward/backward

- Find position of 'R': O(15) → P(16) → Q(17) → R(18) ✓

### 1.4 QUESTION TYPES WITH DETAILED SOLUTIONS

#### TYPE A: Direct Position Finding

**Concept:** When asked "Xth to the left/right of Yth from left/right"

**Rule:**
- Left = Subtract
- Right = Add
- From Left = Count from beginning
- From Right = Count from end

**Solved Example 1:**
Q: Which letter is 5th to the right of 10th letter from left?

**Step-by-step:**
1. 10th from left = J (position 10)
2. 5th to right = Add 5: 10 + 5 = 15
3. 15th letter = O
4. **Answer: O**

**Solved Example 2:**
Q: Which letter is 7th to the left of 12th letter from right?

**Step-by-step:**
1. 12th from right = Total 26 - 12 + 1 = 15th from left
2. 7th to left = 15 - 7 = 8th from left
3. 8th letter = H
4. **Answer: H**

**Shortcut Formula:**
```
Final Position = Initial Position ± Shift
(Use + for right, - for left)
```

#### TYPE B: Middle Letter Between Two Letters

**Formula:** Position = (Position₁ + Position₂) ÷ 2

**Solved Example:**
Q: Which letter is exactly midway between G and T?

**Solution:**
1. G = 7, T = 20
2. (7 + 20) ÷ 2 = 27 ÷ 2 = 13.5
3. 13.5 means between 13 and 14
4. Letters at 13 and 14 = M and N
5. **Answer: M and N**

**Special Case:** If sum is odd, answer is two letters

#### TYPE C: Number of Letters Between Two Letters

**Formula:** |Position₁ - Position₂| - 1

**Solved Example:**
Q: How many letters between P and V?

**Solution:**
1. P = 16, V = 22
2. |16 - 22| = 6
3. 6 - 1 = 5
4. **Answer: 5 letters (Q, R, S, T, U)**

#### TYPE D: Word Formation from Selected Letters

**Solved Example:**
Q: If from "EDUCATION", 2nd, 4th, 6th, and 8th letters are taken, what meaningful word can be formed?

**Solution:**
1. E D U C A T I O N
   Positions: 1=E, 2=D, 3=U, 4=C, 5=A, 6=T, 7=I, 8=O, 9=N
2. Selected: 2=D, 4=C, 6=T, 8=O
3. Letters: D, C, T, O
4. Rearrange to form: D O C T (Doctor)
5. **Answer: DOCT**

#### TYPE E: Reversing Half Alphabet

**Concept:** When first/last half is reversed, positions change

**Solved Example:**
Q: If first half of alphabet (A-M) is reversed, which letter is 10th from right?

**Solution:**
1. Original first half: A(1) to M(13)
2. Reversed: M(1), L(2), K(3), J(4), I(5), H(6), G(7), F(8), E(9), D(10), C(11), B(12), A(13)
3. Second half (N-Z) unchanged: N(14) to Z(26)
4. 10th from right = Position 26 - 10 + 1 = 17 from left
5. 17th letter = In new arrangement: 1-13 reversed, 14-26 normal
6. 17 > 13, so from normal part: 17-13=4th of second half
7. Second half: N(14), O(15), P(16), Q(17) → Q
8. **Answer: Q**

### 1.5 ADVANCED TRICKS & SHORTCUTS

**Trick 1: Vowel-Consonant Recognition**
- Vowels: A, E, I, O, U (Positions: 1,5,9,15,21)
- Consonants: All others

**Trick 2: Cyclic Difference**
For questions like "How many letters between X and Y when counted in circular manner?"
```
Difference = min(|p-q|, 26-|p-q|) - 1
```

**Trick 3: Position from Right Fast Calculation**
```
Rank from Right = 27 - Rank from Left
(For letters only)
```

### 2.1 LETTER SHIFTING PATTERNS

#### Pattern 1: Uniform Shift (Same for all letters)

**Type A: Forward Shift (+n)**
```
Example: DOG → GRJ
D(4)+3=G(7)
O(15)+3=R(18)
G(7)+3=J(10)
Rule: +3 each letter
```

**Type B: Backward Shift (-n)**
```
Example: ZEBRA → YDAQZ
Z(26)-1=Y(25)
E(5)-1=D(4)
B(2)-1=A(1)
R(18)-1=Q(17)
A(1)-1=Z(26)
Rule: -1 each letter
```

**Type C: Mixed Uniform (Different but constant shift)**
```
Example: CAT → DBU? Let's verify:
C(3)+1=D(4)
A(1)+1=B(2)
T(20)+1=U(21) ✓ All +1
```

#### Pattern 2: Variable Shift (Pattern in differences)

**Type A: Increasing Shift**
```
Example: ABC → BDF
A(1)+1=B(2)

### 2.8 COMPLETE SHORTCUTS FOR CODING

**Shortcut 1: Quick Opposite Calculation**
Instead of 27-x, use:
- A↔Z (1↔26)
- B↔Y (2↔25)
- M↔N (13↔14) [Only consecutive pair]

**Shortcut 2: Pattern Detection**
Look at first and last letters first:
- If first letter changes by +k and last by +k → Uniform shift
- If first changes by +a, last by -b → Mixed pattern

**Shortcut 3: Vowel Position Memory**
Vowels at: 1(A),5(E),9(I),15(O),21(U)
Know these for quick vowel/consonant identification

**Shortcut 4: Difference Method**
To find shift amount:
```
Shift = Code Position - Original Position
(For each letter)
```

If X is father of Y and Z is mother of Y, then X and Z are married

# 📖 SECTION 6: RANKING

## 📊 COMPLETE RANKING GUIDE

### 6.1 FUNDAMENTAL FORMULAS

#### Formula 1: Total Persons in a Row
```
Total = Rank from Left + Rank from Right - 1
```

**Why?** Left rank and right rank count the same person twice, so subtract 1.

#### Formula 2: Finding Rank from Other Side
```
Rank from Right = Total - Rank from Left + 1
Rank from Left = Total - Rank from Right + 1
```

#### Formula 3: Number of Persons Between Two
```
Between = |Rank₁ - Rank₂| - 1
```
(When both ranks are from same side)

OR
```
Between = Total - Rank₁(from left) - Rank₂(from right)
```
(When ranks are from different ends)

#### Formula 4: Position After Interchange
When two persons interchange positions:
- A's new position = B's old position
- B's new position = A's old position

### 6.2 PROBLEM TYPES WITH SOLUTIONS

#### Type 1: Basic Ranking

**Solved Example 1:**
Q: In a row, Rahul is 7th from left and 9th from right. Total students?

**Solution:**
Total = 7 + 9 - 1 = 15
**Answer:** 15 students

**Solved Example 2:**
Q: In a class of 40, Riya is 12th from left. Her rank from right?

**Solution:**
Rank from right = 40 - 12 + 1 = 29
**Answer:** 29th from right

#### Type 2: Persons Between Two

**Solved Example 1 (Same end ranks):**
Q: In a row, A is 8th from left, B is 15th from left. How many between them?

**Solution:**
Between = |8 - 15| - 1 = 7 - 1 = 6
**Answer:** 6 persons

**Solved Example 2 (Different ends):**
Q: In a row of 30, A is 8th from left, B is 12th from right. How many between them?

**Method 1:**
A's rank from right = 30 - 8 + 1 = 23rd from right
Between = |23 - 12| - 1 = 11 - 1 = 10

**Method 2 (Shortcut):**
Between = Total - Left rank - Right rank
= 30 - 8 - 12 = 10

**Answer:** 10 persons

#### Type 3: Interchanging Positions

**Solved Example:**
Q: In a row, A is 9th from left, B is 11th from right. They interchange positions. Now A is 15th from left. Find:
a) Total students
b) New position of B from right

**Solution:**

**Step 1:** After interchange, A's new position = B's old position
So B was 15th from left originally

**Step 2:** B is 11th from right originally
Total = Left rank + Right rank - 1 = 15 + 11 - 1 = 25

**Step 3:** After interchange, B takes A's old position
A was 9th from left originally
So B is now 9th from left

**Step 4:** B's new rank from right = Total - New left rank + 1
= 25 - 9 + 1 = 17

**Answer:** Total = 25, B is 17th from right

#### Type 4: Ranking with Overlap

**Solved Example:**
Q: In a row of 50, A is 20th from left, B is 25th from right. How many are between them if they are on same side? If they overlap?

**Solution:**
Between formula = Total - Left - Right = 50 - 20 - 25 = 5

Since result is positive (5), they are separate with 5 persons between them.

If result were negative, they would overlap:
Example: If total = 30, A=20 from left, B=15 from right
Between = 30 - 20 - 15 = -5 (negative means overlap of 5 persons)

**Answer:** 5 persons between them

#### Type 5: Ranking in Both Directions

**Solved Example:**
Q: In a row of 50, if A is ahead of B by 7 places, and A is 15th from left, what is B's position from right?

**Solution:**
1. A is 15th from left
2. A ahead of B by 7 places → B is 7 places behind A
3. So B's rank from left = 15 + 7 = 22
4. B's rank from right = 50 - 22 + 1 = 29

**Answer:** 29th from right

#### Type 6: Minimum/Maximum Persons

**Solved Example:**

**Solution:**

If A is 5th from left, at least 5 persons. If B is 8th from right, at least 8 persons.

If they are adjacent: A at 5 from left, B at 8 from right means total = 5 + 8 = 13? Wait:

Better: For minimum, place them with no one between but not overlapping:

For maximum: They should be as far apart as possible
Maximum = Left rank + Right rank - 1 + (something)? Actually maximum is unbounded theoretically, but in given context, if no other constraints, maximum is large.

#### Type 7: Ranking with Conditions

**Solved Example:**
Q: In a row of 40 students, Ramesh is 8th from left and Suresh is 12th from right. If 5 students are between Ramesh and Suresh, how many students are to the left of Ramesh?

**Solution:**
Total = 40, Ramesh left = 8, Suresh right = 12

Suresh pos from left = Total - Right + 1 = 40 - 12 + 1 = 29

### 6.3 SHORTCUTS & TRICKS

**Trick 1: Between Formula Memory**
When ranks from different ends: Between = Total - L - R
When ranks from same end: Between = |L₁ - L₂| - 1

**Trick 2: Interchange Shortcut**
After interchange:
- New position of A = Old position of B
- New position of B = Old position of A
- Total remains same

**Trick 3: Finding Rank from Other End**
Rank from Right = Total + 1 - Rank from Left
(Works for any total)

**Trick 4: When "Ahead/Behind" Given**
If A is ahead of B by k places:
B's rank = A's rank + k (if ahead means higher rank number? Be careful)
Usually "ahead" means smaller rank number (closer to front)

# 📖 SECTION 7: SYLLOGISMS

### 7.1 THE FOUR STATEMENT TYPES

#### Type A: Universal Affirmative (All S are P)
- **Form:** All A are B
- **Venn Diagram:** Circle A completely inside circle B
- **Implications:** 
  - Some A are B (True)
  - Some B are A (True)
  - No definite about A not being something

```

    │  ┌───┐  │
    │  │ A │  │

    └─────────┘
```

#### Type E: Universal Negative (No A are B)
- **Form:** No A are B
- **Venn Diagram:** Two separate circles with no overlap
- **Implications:**
  - No B are A (True)
  - Some A are not B (True)
  - Some B are not A (True)

```
┌───┐    ┌───┐
│ A │    │ B │

```

#### Type I: Particular Affirmative (Some A are B)
- **Form:** Some A are B
- **Venn Diagram:** Overlapping circles with shaded intersection

  - Some B are A (True)

- **Form:** Some A are not B
- **Venn Diagram:** Part of A outside B
- **Implications:**

### 7.2 VENN DIAGRAM RULES

#### Rule 1: All A are B
```
Possible conclusions:
✓ Some A are B
✓ Some B are A
✗ Some A are not B (False)
✗ No A are B (False)
```

#### Rule 2: No A are B
```
Possible conclusions:
✓ No B are A
✓ Some A are not B
✓ Some B are not A
✗ Some A are B (False)
✗ All A are B (False)
```

#### Rule 3: Some A are B
```
Possible conclusions:
✓ Some B are A
✗ All A are B (Not necessarily)
✗ No A are B (False)
✗ Some A are not B (Not necessarily)
```

#### Rule 4: Some A are not B
```
Possible conclusions:
✗ All A are B (False)
✗ No A are B (Not necessarily)
✗ Some B are not A (Not necessarily)
```

### 7.3 THE THREE-STATEMENT SYLLOGISMS

#### Method: Draw All Possible Venn Diagrams

**Solved Example 1:**
Statements:
1. All cats are animals
2. Some animals are pets

Conclusions:
I. Some cats are pets
II. Some animals are cats

**Step 1: Draw first statement**
All cats are animals → Circle cats inside animals

**Step 2: Add second statement**
Some animals are pets → Overlap between animals and pets
This overlap could be:
- In the cats region (some cats are pets)
- In the animals but outside cats region (some animals that are not cats are pets)

**Step 3: Check conclusions**
I. Some cats are pets? Not necessarily (pets could be only non-cat animals) → Doesn't follow
II. Some animals are cats? From "All cats are animals", definitely some animals (the cats) are cats → Follows

**Answer:** Only II follows

**Solved Example 2:**
Statements:
1. No pen is pencil
2. Some pencils are erasers

Conclusions:
I. Some pens are not erasers
II. Some erasers are not pens

**Step 1: Draw first statement**
No pen is pencil → Two separate circles

**Step 2: Add second statement**
Some pencils are erasers → Overlap between pencils and erasers
Erasers could be:
- Only overlapping with pencils (no contact with pens)
- Also overlapping with pens? But pens and pencils are separate, so erasers can overlap with pens only if erasers extend beyond pencils

**Step 3: Check conclusions**
I. Some pens are not erasers? Pens could be completely separate from erasers, or could overlap. But we don't know. So "some pens are not erasers" is definitely true? Wait, if pens overlap with erasers, then some pens ARE erasers, so "some pens are not erasers" would still be true as long as not ALL pens are erasers. But we don't know if any pens are erasers. Actually, we can draw a diagram where all pens are erasers (possible if erasers circle contains pens completely). In that case, "some pens are not erasers" would be false. So it doesn't follow necessarily.

II. Some erasers are not pens? Since pencils are not pens, and some erasers are pencils, those erasers are definitely not pens. So yes, some erasers (the ones that are pencils) are not pens. This follows.

**Answer:** Only II follows

### 7.4 COMPLEMENTARY PAIR CONCEPT

**Definition:** Two conclusions that together cover all possibilities, but neither is individually certain

**Complementary Pairs:**
1. "Some A are B" and "No A are B" (Cannot both be false)
2. "Some A are B" and "Some A are not B" (Cannot both be false)
3. "All A are B" and "Some A are not B" (Cannot both be false)

**Rule:** If neither conclusion follows individually, but they form a complementary pair, then "Either I or II follows"

**Solved Example:**
Statements:
1. All dogs are animals
2. No animal is stone

Conclusions:
I. Some dogs are not stones
II. No dog is stone

**Solution:**
From statements: All dogs are animals, and no animal is stone → Therefore no dog is stone (II follows directly)
So I is also true but weaker. Actually both follow, but II is stronger.

Better complementary example:
Statements: Some pens are pencils
Conclusions: I. All pens are pencils, II. No pen is pencil

These are complementary? No, they can both be false (if some but not all pens are pencils). So not complementary.

True complementary:
Statements: No direct relation between A and B
Conclusions: I. Some A are B, II. No A are B
These are complementary because in any possible world, one of them must be true (either there is some overlap or no overlap). If neither follows individually, then "Either I or II follows".

### 7.5 POSSIBILITY CONCLUSIONS

**Concept:** When conclusion contains words like "can be", "possible", "may be", we only need to check if a diagram exists that satisfies both statements and the conclusion

**Solved Example:**
Statements:
1. All A are B
2. Some B are C

Conclusions:
I. Some A are C (Possibility)
II. All A are C (Possibility)

**Solution:**
For I: Can we draw a diagram where some A are C? Yes, by making the A-C overlap exist.
For II: Can we draw a diagram where all A are C? Yes, by putting A completely inside C (which is inside B)

Both are possible, so both possibility conclusions follow.

**Answer:** Both I and II follow (as possibilities)

### 7.6 THE DEDUCTION RULES

#### Rule 1: The Chain Rule
If All A are B and All B are C, then All A are C

#### Rule 2: The No-Chain Rule
If All A are B and No B are C, then No A are C

#### Rule 3: The Some Chain
If Some A are B and All B are C, then Some A are C

#### Rule 4: The Some-No Conflict
If Some A are B and No B are C, then Some A are not C

**Solved Example (Complex):**
Statements:

# 📖 SECTION 8: INEQUALITY

## ⚖️ COMPLETE INEQUALITY GUIDE

### 8.1 SYMBOL MEANINGS

| Symbol | Meaning | Example |
|--------|---------|---------|
| > | Greater than | A > B means A is greater than B |
| < | Less than | A < B means A is less than B |
| = | Equal to | A = B means A equals B |
| ≥ | Greater than or equal to | A ≥ B means A ≥ B (A is at least B) |
| ≤ | Less than or equal to | A ≤ B means A ≤ B (A is at most B) |
| ≠ | Not equal to | A ≠ B means A is not equal to B |

### 8.2 BASIC RULES

#### Rule 1: Transitive Property
If A > B and B > C, then A > C
If A < B and B < C, then A < C
If A ≥ B and B ≥ C, then A ≥ C
If A ≤ B and B ≤ C, then A ≤ C

#### Rule 2: Combining with Equality
If A ≥ B and B = C, then A ≥ C
If A > B and B = C, then A > C
If A = B and B ≥ C, then A ≥ C

#### Rule 3: When Symbols Change Direction
If A > B < C → No relation between A and C
If A < B > C → No relation between A and C
If A ≥ B ≤ C → No relation between A and C

### 8.3 THE CHAIN RULE

**Rule:** If all inequality symbols face the same direction, we can combine them into a chain

**Solved Example 1:**
A > B > C > D
Conclusion: A > D ✓

**Solved Example 2:**
P ≥ Q ≥ R = S < T
Can we say P ≥ S? Yes (P ≥ Q ≥ R = S → P ≥ S)
Can we say P > S? No (P could equal S)
Can we say T > R? Yes (R = S < T → R < T)

### 8.4 PROBLEM TYPES

#### Type 1: Direct Inequality Chain

**Solved Example:**
Statements: P ≥ Q, Q > R, R = S, S < T
Conclusions: I. P > R, II. T > R

**Solution:**
Combine: P ≥ Q > R = S < T

I. P > R? From P ≥ Q > R → P > R (even if P=Q, Q>R so P>R) ✓
II. T > R? From R = S < T → R < T ✓

**Answer:** Both follow

#### Type 2: Coded Inequality

**Solved Example:**
@ means ≥
# means ≤
$ means >
% means <
& means =

Statements: A @ B, B $ C, C & D, D % E
Conclusions: I. A $ C, II. D # B

**Solution:**
Decode: A ≥ B, B > C, C = D, D < E
So: A ≥ B > C = D < E

I. A $ C means A > C? From A ≥ B > C → A > C ✓
II. D # B means D ≤ B? From B > C = D → B > D, so D < B, thus D ≤ B ✓

**Answer:** Both follow

#### Type 3: Complementary Pair in Inequality

**Concept:** When conclusions are like A > B and A = B, they are complementary (cover all possibilities except A < B)

**Rule:** If neither follows individually but they form a complementary pair, then "Either I or II follows"

**Solved Example:**
Statements: P ≥ Q, Q ≤ R, R > S
Conclusions: I. P > S, II. P = S

**Solution:**
From P ≥ Q ≤ R > S, no direct P-S relation
But could P > S be true? Possibly if chain works out
Could P = S be true? Possibly
But are they complementary? No, because they don't cover P < S possibility
So neither follows, and no "either-or"

**Better Example:**
Statements: A ≥ B, B ≥ C
Conclusions: I. A > C, II. A = C

Here, from A ≥ B ≥ C, we know A ≥ C
So either A > C or A = C must be true
But individually, neither is certain
Thus "Either I or II follows"

**Answer:** Either I or II follows

#### Type 4: Multiple Statements with Possibilities

**Solved Example:**
Statements: A > B, B ≥ C, C < D, D = E
Conclusions: I. A > C, II. A = C, III. A > E, IV. A = E

**Solution:**
From A > B ≥ C → A > C (I true, II false)
From A > B ≥ C < D = E, no direct A-E relation
Could A > E? Possibly if chain allows
Could A = E? Possibly
But they don't cover all possibilities (A < E possible)
So III and IV don't follow individually, and no either-or

**Answer:** Only I follows

### 8.5 SHORTCUTS & TRICKS

**Trick 1: The ≥ and > Difference**
When combining, note:
- A ≥ B > C → A > C (definite)
- A > B ≥ C → A > C (definite)
- A ≥ B ≥ C → A ≥ C (not necessarily >)

**Trick 2: Reverse Direction**
To compare in opposite direction, reverse all symbols and flip inequality:
If A > B, then B < A
If A ≥ B, then B ≤ A

**Trick 3: Substitution**
If two variables are equal, substitute one for the other anywhere

**Trick 4: The Either-Or Condition**
Either I or II follows if:
1. Both conclusions have same variables
2. One is >, one is = (or one ≥, one ≤ etc)
3. They cover all possibilities except the opposite inequality

Common complementary pairs:
- A > B and A = B (cover A ≥ B)
- A < B and A = B (cover A ≤ B)
- A ≥ B and A ≤ B (cover all, but these can both be true when A=B)

# 📖 SECTION 9: INPUT-OUTPUT

## 🤖 COMPLETE MACHINE INPUT-OUTPUT GUIDE

### 9.1 WHAT IS INPUT-OUTPUT?

A machine rearranges numbers and/or words in a step-by-step pattern. Each step follows a specific rule until the final arrangement is reached. You need to identify the pattern and predict steps for new input.

### 9.2 COMMON PATTERNS

#### Pattern 1: Ascending/Descending Order (Sorting)

**Example:**
Input: 42 36 17 29 53 11
Step 1: 11 42 36 17 29 53 (smallest moved to left)
Step 2: 11 17 42 36 29 53 (next smallest)
Step 3: 11 17 29 42 36 53
Step 4: 11 17 29 36 42 53
Step 5: 11 17 29 36 42 53 (sorted)

**Pattern:** In each step, the next smallest number is placed in its correct position from left

#### Pattern 2: Descending Order

**Example:**
Input: 42 36 17 29 53 11
Step 1: 53 42 36 17 29 11 (largest to left)
Step 2: 53 42 36 17 29 11? Wait, next largest? Let's check:
Actually better example:
Input: 15 28 13 42 31
Step 1: 42 15 28 13 31
Step 2: 42 31 15 28 13
Step 3: 42 31 28 15 13
Step 4: 42 31 28 15 13 (sorted descending)

#### Pattern 3: Shifting Extremes

**Example:**
Input: 85 16 34 72 41 28
Step 1: 16 85 34 72 41 28 (smallest to left)
Step 2: 16 28 85 34 72 41 (next smallest to 2nd)
Step 3: 16 28 34 85 72 41
Step 4: 16 28 34 41 85 72
Step 5: 16 28 34 41 72 85

#### Pattern 4: Alternating Word-Number

**Example:**
Input: 42 Apple 36 Ball 17 Cat
Step 1: Apple 42 36 Ball 17 Cat (word comes to front)
Step 2: Apple Ball 42 36 17 Cat (next word)
Step 3: Apple Ball Cat 42 36 17
Step 4: Apple Ball Cat 17 42 36 (smallest number after words)
Step 5: Apple Ball Cat 17 36 42

**Pattern:** First arrange words alphabetically, then numbers ascending

#### Pattern 5: Based on Digit Sum/Value

**Example:**
Input: 56, 43, 72, 38, 91
Digit sums: 56(11), 43(7), 72(9), 38(11), 91(10)
Step 1: 43 56 72 38 91 (by digit sum ascending? 7,9,10,11,11)
Step 2: 43 72 56 38 91
Step 3: 43 72 91 56 38
Step 4: 43 72 91 38 56 (within same digit sum, original order preserved?)

#### Pattern 6: Interchanging Pairs

**Example:**
Input: A B C D E F
Step 1: B A D C F E (swap adjacent pairs: 1-2, 3-4, 5-6)
Step 2: A B C D E F (back to original? No, pattern might continue differently)

Better:
Input: 1 2 3 4 5 6
Step 1: 2 1 4 3 6 5
Step 2: 2 4 1 6 3 5 (more complex)

#### Pattern 7: Based on Number Properties

**Example (Even-Odd):**
Input: 23 46 18 35 52 27
Step 1: 46 23 18 35 52 27 (even numbers first)
Step 2: 46 18 23 35 52 27
Step 3: 46 18 52 23 35 27 (all evens sorted, then odds)
Step 4: 46 18 52 23 27 35 (odds sorted)

### 9.3 STEP-BY-STEP PATTERN IDENTIFICATION

**Methodology:**

**Step 1:** Compare Input with Step 1
- Which element moved?

# 📖 SECTION 10: CLOCK & CALENDAR

## ⏰ COMPLETE TIME GUIDE

### 10.1 CLOCK FUNDAMENTALS

#### Basic Facts
- Clock is a circle: 360°
- Hour hand moves 360° in 12 hours = 30° per hour = 0.5° per minute
- Minute hand moves 360° in 60 minutes = 6° per minute
- Relative speed of minute hand w.r.t hour hand = 6° - 0.5° = 5.5° per minute

#### Important Values
| Time | Hour Hand Position | Minute Hand Position | Angle |
|------|-------------------|---------------------|-------|
| 3:00 | 90° | 0° | 90° |
| 6:00 | 180° | 0° | 180° |
| 9:00 | 270° | 0° | 90° |
| 12:00 | 0° | 0° | 0° |

### 10.2 FORMULAS FOR CLOCK

#### Formula 1: Angle Between Hands
```
Angle = |30H - 5.5M|
Where H = hours (in 12-hour format), M = minutes
```

**Note:** If angle > 180°, take 360° - angle (smaller angle)

**Solved Example 1:**
Q: Find angle at 3:30
Angle = |30×3 - 5.5×30| = |90 - 165| = 75°
**Answer:** 75°

**Solved Example 2:**
Q: Find angle at 4:20
Angle = |30×4 - 5.5×20| = |120 - 110| = 10°
**Answer:** 10°

**Solved Example 3:**
Q: Find angle at 7:40
Angle = |30×7 - 5.5×40| = |210 - 220| = 10°
Wait, 210-220 = -10, absolute = 10°
**Answer:** 10°

#### Formula 2: Time When Hands Coincide (Angle = 0°)
```
30H - 5.5M = 0
M = (30H)/5.5 = (60H)/11
```

**Solved Example:**
Q: Between 4 and 5, when do hands coincide?
M = (60×4)/11 = 240/11 = 21.81 minutes = 21 minutes 49 seconds (approx)
**Answer:** 4:21:49

#### Formula 3: Time When Hands are Opposite (180°)
```
30H - 5.5M = 180  OR  30H - 5.5M = -180
```

**Solved Example:**
Q: Between 5 and 6, when are hands opposite?
30×5 - 5.5M = 180
150 - 5.5M = 180
-5.5M = 30
M = -30/5.5 = -5.45 (not valid)
Try other equation: 150 - 5.5M = -180
-5.5M = -330
M = 330/5.5 = 60 minutes
So at 5:60 = 6:00, which is actually between 5 and 6? 5:60 is 6:00, so at 6:00 exactly they are opposite.
**Answer:** 6:00

#### Formula 4: Time When Hands are at Right Angle (90°)
```
30H - 5.5M = 90  OR  30H - 5.5M = -90  OR  30H - 5.5M = 270 (for 90° the other way)
```

### 10.3 SPECIAL CASES

#### Case 1: Both Hands Coincide
They coincide 11 times in every 12-hour period
- At approximately 1:05, 2:10, 3:16, 4:21, 5:27, 6:32, 7:38, 8:43, 9:49, 10:54, 12:00

#### Case 2: Mirror Image Time
For horizontal mirror (reflection in mirror):
```
Mirror Time = 12:00 - Given Time
(Subtract from 11:60 if needed)
```

**Solved Example:**
Q: Find mirror image of 4:30
12:00 - 4:30 = 7:30
**Answer:** 7:30

**Check:** At 4:30, hour hand between 4 and 5, minute hand at 6. In mirror, hour hand between 7 and 8, minute hand at 6 → 7:30 ✓

#### Case 3: Water Image Time
For water image (vertical mirror), formula is:
```
Water Time = 18:30 - Given Time
(For 12-hour format, convert to 24-hour first)
```

**Solved Example:**
Q: Find water image of 4:30
18:30 - 4:30 = 14:00 = 2:00
**Answer:** 2:00

### 10.4 CALENDAR FUNDAMENTALS

#### Odd Days Concept
- Ordinary year: 365 days = 52 weeks + 1 day → 1 odd day
- Leap year: 366 days = 52 weeks + 2 days → 2 odd days

#### Leap Year Rules
1. Year divisible by 4 → Leap year (except century years)
2. Century years divisible by 400 → Leap year
3. Examples:
   - 1900: Not leap (century not divisible by 400)
   - 2000: Leap (divisible by 400)
   - 2024: Leap (divisible by 4)

#### Day Codes
```
Sunday = 0
Monday = 1
Tuesday = 2
Wednesday = 3
Thursday = 4
Friday = 5
Saturday = 6
```

#### Month Codes (For Zeller's Rule type calculations)

|-------|------|----------------|

| February | 3 | 2 |
| March | 3 | 3 |
| April | 6 | 6 |
| May | 1 | 1 |
| June | 4 | 4 |
| July | 6 | 6 |

### 10.5 CALENDAR FORMULAS

#### Formula 1: Zeller's Rule (Simplified)
```
Day = (Date + Month Code + Year Code + Century Code) mod 7
```
Where:
- Year Code = (Last 2 digits + floor(Last 2 digits/4)) mod 7
- For January/February in leap year, use leap year month codes

**Solved Example 1:**
Q: What day was 15th August 1947?

**Solution:**
Date = 15
Month Code for August = 2
Year Code = 47 + floor(47/4) = 47 + 11 = 58 mod 7 = 2 (since 56 is multiple of 7, remainder 2)
Century Code for 1900s = 0

Sum = 15 + 2 + 2 + 0 = 19 mod 7 = 5
Day code 5 = Friday

**Answer:** Friday

**Solved Example 2:**
Q: What day was 26th January 1950?

**Solution:**
Date = 26
Month Code for January = 0 (not leap year, 1950 not leap)
Year Code = 50 + floor(50/4) = 50 + 12 = 62 mod 7 = 62 - 56 = 6
Century Code = 0

Sum = 26 + 0 + 6 + 0 = 32 mod 7 = 4
Day code 4 = Thursday

**Answer:** Thursday

#### Formula 2: Odd Days Method

**Solved Example:**
Q: What day was 15th August 1947? (Using odd days)

**Step 1:** Find odd days from 1900 to 1946
- Years = 47 years (1900-1946)
- Leap years in this period: 1904,1908,...,1944
- Number of leap years = floor(47/4) = 11 (but exclude 1900? 1900 not leap, so formula works)
- Actually count: 1904 to 1944 step 4 = (1944-1904)/4 +1 = 40/4+1=10+1=11 ✓
- Ordinary years = 47 - 11 = 36
- Odd days = 36×1 + 11×2 = 36 + 22 = 58
- 58 mod 7 = 2 (since 56 is multiple of 7)

**Step 2:** Odd days in 1947 till August 15
- Jan(31) = 3 odd days (31 mod 7 = 3)
- Feb(28) = 0 odd days
- Mar(31) = 3
- Apr(30) = 2
- May(31) = 3
- Jun(30) = 2
- Jul(31) = 3
- Aug(15) = 15 days = 1 odd day (14 days = 2 weeks)
- Total = 3+0+3+2+3+2+3+1 = 17
- 17 mod 7 = 3

**Step 3:** Total odd days = 2 + 3 = 5
Day code 5 = Friday

**Answer:** Friday

### 10.6 SPECIAL CALENDAR CONCEPTS

#### Concept 1: Same Calendar Year
A year has the same calendar as another year if:
1. Both are ordinary or both leap
2. The odd days difference is multiple of 7

**Example:** 2023 (ordinary, 1 odd day) repeats in 2034? 2024 leap (2 odd), 2025(1), 2026(1), 2027(1), 2028(2), 2029(1), 2030(1), 2031(1), 2032(2), 2033(1), 2034(1). Sum odd days from 2024 to 2033 = 2+1+1+1+2+1+1+1+2+1 = 13 mod 7 = 6, plus 2023's 1 = 7 mod 7 = 0. So 2034 has same calendar as 2023.

#### Concept 2: Number of Times Hands Overlap
In 12 hours, hands overlap 11 times
In 24 hours, they overlap 22 times

#### Concept 3: Angle Between Hands Shortcut
For quick estimation:
- At H:00, angle = 30H
- Every minute, angle changes by 5.5°

# 📖 SECTION 11: DICE & CUBES

## 🎲 COMPLETE DICE AND CUBE GUIDE

### 11.1 TYPES OF DICE

#### Type 1: Standard Dice
- Opposite faces sum to 7
- Configuration:
  ```
  1 opposite 6
  2 opposite 5
  3 opposite 4
  ```
- This is the most common type in exams

#### Type 2: Ordinary Dice
- Opposite faces may not sum to 7
- Need to deduce from given positions

### 11.2 RULES FOR OPPOSITE FACES

#### Rule 1: In any two positions, if two faces are common and in same orientation, the remaining faces are opposite

**Solved Example:**
Position 1: Top=1, Front=2, Right=3
Position 2: Top=1, Front=2, Right=4
Then 3 and 4 are opposite

#### Rule 2: If a face is adjacent to another face in all given positions, they cannot be opposite

#### Rule 3: In a single position, faces shown are all adjacent to each other

### 11.3 PROBLEM TYPES FOR DICE

#### Type 1: Finding Opposite from Standard Dice

**Solved Example:**
Q: In a standard dice, if top is 2 and front is 3, what is opposite of 2?

**Solution:**
In standard dice, opposite of 2 is 5
(Doesn't depend on orientation)

**Answer:** 5

#### Type 2: From Multiple Positions

**Solved Example:**
Q: Two positions of a dice are shown:
Position A: Top=3, Front=4, Right=6
Position B: Top=3, Front=1, Right=2
Find opposite of 4.

**Solution:**
In both positions, 3 is common on top
Faces adjacent to 3: In A: 4,6; In B: 1,2
Thus 4,6,1,2 are all adjacent to 3
So remaining face (5) must be opposite to 3

Now, 4 is adjacent to 3 and 6 (from A)
4 never appears with 1,2,5 in any position
So 4 could be opposite to either 1,2, or 5
But we know 5 is opposite to 3, so 5 is not opposite to 4
So 4 is opposite to either 1 or 2

From B, 1 and 2 are adjacent to each other (both adjacent to 3 and shown together)
So 1 and 2 cannot be opposite each other
Thus 4 must be opposite to one of them
Without more info, cannot determine which

**Answer:** Cannot determine uniquely

#### Type 3: Construction from Net

**Solved Example:**
Q: Which number is opposite to 2 in this net?
```
    ┌───┐
    │ 1 │
┌───┼───┼───┐
│ 4 │ 2 │ 5 │
└───┼───┼───┘
    │ 3 │
    └───┘
    │ 6 │
    └───┘
```

**Solution:**
In a cube net, opposite faces are those that are separated by one face
- 2 is adjacent to 1,4,5,3 (shares edges)
- The only face not adjacent to 2 is 6
- Therefore 2 and 6 are opposite

**Answer:** 6

#### Type 4: Finding Adjacent Faces

**Solved Example:**
Q: In a dice, if 1 is opposite to 6, 2 is opposite to 5, and 3 is opposite to 4, which faces are adjacent to 1?

**Solution:**
All faces except 6 are adjacent to 1
So 2,3,4,5 are adjacent to 1

**Answer:** 2,3,4,5

### 11.4 CUBE PAINTING PROBLEMS

#### Basic Formula for Painted Cube

For a cube of side 'n' (painted on all faces) cut into n³ small cubes:

| Painting Type | Formula | Description |
|---------------|---------|-------------|
| 3 faces painted | 8 | Corner pieces (always 8) |
| 2 faces painted | 12(n-2) | Edge pieces excluding corners |
| 1 face painted | 6(n-2)² | Face centers |
| 0 faces painted | (n-2)³ | Interior cubes |
| Total | n³ | Sum of all |

**Solved Example 1:**
Q: A cube of side 4 is painted red on all faces and cut into 64 small cubes. How many have exactly 2 faces painted?

**Solution:**
n = 4
2 faces painted = 12(n-2) = 12×2 = 24
**Answer:** 24

**Solved Example 2:**
Q: A cube of side 5 is painted on all faces and cut into 125 cubes. How many have no face painted?

**Solution:**
n = 5
0 faces painted = (n-2)³ = 3³ = 27
**Answer:** 27

#### Variations in Painting

**Type A: Different Colors on Different Faces**

**Solved Example:**
Q: A cube of side 4 is painted red on top and bottom, green on front and back, blue on left and right. How many cubes have at least two colors?

**Solution:**
This requires careful counting of edges and corners:
- Corner cubes (8) have 3 colors each
- Edge cubes (excluding corners) have 2 colors
- Face center cubes have 1 color
- Interior have 0

But need to count by position.

**Type B: Only Some Faces Painted**

**Solved Example:**
Q: A cube of side 5 is painted only on one face. How many cubes have paint?

**Solution:**
One face has 5×5 = 25 cubes
All these have at least one face painted
**Answer:** 25

### 11.5 CUBE CUTTING PROBLEMS

#### Formula for Number of Cuts

To cut a cube into n³ smaller cubes:
- Number of cuts = 3(n-1) (if cutting all dimensions equally)

**Solved Example:**
Q: How many cuts are needed to cut a cube into 125 smaller cubes?

**Solution:**
125 = 5³, so n=5
Cuts = 3(5-1) = 3×4 = 12 cuts
**Answer:** 12

### 11.6 SHORTCUTS & TRICKS

**Trick 1: Opposite Faces in Standard Dice**
Just remember: 1-6, 2-5, 3-4

**Trick 2: From Two Positions**
If two faces are common, the third face in each position are opposite

**Trick 3: From Net**
In a net, faces that are separated by exactly one face in the layout are opposite

**Trick 4: Painted Cube Memory**
For n=3: 3-faced=8, 2-faced=12, 1-faced=6, 0-faced=1
For n=4: 3-faced=8, 2-faced=24, 1-faced=24, 0-faced=8
